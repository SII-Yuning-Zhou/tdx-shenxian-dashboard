# SignalBoard Live

SignalBoard Live 是一个面向 A 股盘中观察的量化实盘信号看板，用于把策略筛选、持仓监控、题材研究和盘中节奏统一展示在一个可远程访问的网页中。

它的定位不是交易下单系统，而是一个实盘研究与风险提醒工具：让使用者在开盘期间更快看到新的候选信号、持仓变化和相关题材背景，减少在多个软件和文件之间来回切换。

## 核心能力

- 盘中信号看板：集中展示策略触发的转势信号、最新价格、均线状态和信号强度。
- 实时与固定周期并行：同时保留盘中实时监控和 60 分钟完成周期确认，便于区分“正在形成的机会”和“已经确认的节点”。
- 持仓风险提醒：针对真实持仓独立监控转弱信号，帮助及时发现需要关注的仓位。
- 研报题材增强：为候选股票补充主营业务、行业标签、题材线索和概念关键词，让信号不只是一个代码。
- 行业颜色分组：相近行业和题材使用一致的视觉标识，方便快速扫视当日主线。
- 远程看板访问：支持在手机、平板或其他电脑上查看盘中看板，适合个人或小团队同步观察。
- 访问保护：看板内容默认需要授权访问，适合展示给指定人员而不是完全公开展示。

## 适合场景

- 开盘期间跟踪策略候选股变化。
- 观察 60 分钟级别的转红、转绿信号。
- 把短线信号和研报题材信息放在同一个界面里判断。
- 对真实持仓做独立的风险提醒。
- 在外出或不在交易主机前时，通过网页查看实盘状态。

## 看板内容

主页看板会按实盘节奏展示几类信息：

- 转红检测：策略监控池中出现的潜在买点或关注点。
- 固定周期确认：已经走完的 60 分钟节点信号。
- 持仓转绿提醒：只针对当前真实持仓的风险信号。
- 真实持仓列表：用于核对当前监控范围。
- 题材解释：主营业务、行业归属、研报线索和概念标签。

这些信息会尽量按最新时间排序，让新出现的信号优先被看到。

## 设计原则

- 实盘优先：盘中扫描不应被网页生成或远程同步拖慢。
- 轻量访问：网页端只负责展示，不承担复杂交易逻辑。
- 信息压缩：同一屏内尽量呈现代码、名称、信号、指标和题材背景。
- 本地可控：核心策略和数据处理运行在本地环境中，远程页面只作为展示层。
- 可扩展：后续可以继续加入日线信号、更多周期、行业热度和回测回放视图。

## 风险说明

本项目仅用于量化研究、盘中观察和信号记录，不构成任何投资建议，也不保证信号收益。市场波动、数据延迟、模型假设和执行条件都会影响实际结果。使用者应结合自身交易计划、风险承受能力和独立判断使用。

## 当前状态

项目处于持续迭代阶段，重点优化方向包括：

- 更稳定的盘中周期 K 线生成。
- 更清晰的买卖信号分层。
- 更完整的题材和主营业务解释。
- 更适合手机端查看的布局。
- 更高可靠性的远程同步和异常恢复。
